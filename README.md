
# DFA Simulator

本项目是编译器设计专题实验一的实现内容，主要完成了 DFA（确定有限自动机）的模拟运行、字符串识别、合法字符串生成，以及一个简单的 DFA 可视化静态网页。

项目包含两个部分：

1. `DFA.cpp`：C++ 实现的 DFA 模拟程序  
2. `index.html`：DFA 可视化静态网页  

---

## 项目结构

```text
root/
├── DFA.cpp              # C++版 DFA 模拟程序
├── index.html           # DFA 可视化静态网页
└── example/
    └── *.dfa            # DFA 输入文件示例
````

---

## 功能说明

### C++ 程序功能

`DFA.cpp` 实现了以下功能：

* 从 `.dfa` 文件读取 DFA 五元组
* 检查 DFA 定义是否合法
* 输入最大长度 `N`，输出语言集中长度 `<= N` 的所有字符串
* 输入字符串，模拟 DFA 的状态转移过程
* 判断输入字符串是否被 DFA 接受

---

### 静态网页功能

`index.html` 实现了 DFA 的可视化展示，主要功能包括：

* 输入 DFA 定义
* 绘制 DFA 状态转移图
* 输入最大长度 `N`，展示长度 `<= N` 的合法字符串
* 输入字符串，判断是否被 DFA 接受

网页为纯静态页面，可直接用浏览器打开运行。

## DFA 输入文件格式

`.dfa` 文件采用简化格式，不使用额外字段前缀。

格式如下：

```text
字符集
状态集
开始状态
接受状态集
状态转移矩阵
```

例如：

```text
a b
1 2
1
2
2 1
2 1
```

含义如下：

* 字符集：`a b`
* 状态集：`1 2`
* 开始状态：`1`
* 接受状态：`2`
* 转移矩阵：

| 当前状态 | a | b |
| ---- | - | - |
| 1    | 2 | 1 |
| 2    | 2 | 1 |

其中，转移矩阵省略行列标签：

* 第 `i` 行对应第 `i` 个状态
* 第 `j` 列对应第 `j` 个输入字符

所以：

```text
2 1
2 1
```

表示：

```text
状态1 输入a -> 状态2
状态1 输入b -> 状态1
状态2 输入a -> 状态2
状态2 输入b -> 状态1
```

该 DFA 可识别所有以 `a` 结尾的字符串。

## 使用示例

运行程序后，输入 DFA 文件路径：

```text
example/example1.dfa
```

然后输入最大长度 `N`：

```text
3
```

程序会输出长度不超过 `3` 的所有合法字符串。

之后输入一个待识别字符串，例如：

```text
aba
```

程序会输出识别过程：

```text
开始状态: 1
1 --a--> 2
2 --b--> 1
1 --a--> 2
结束状态: 2
该字符串被接受
```

## 实验说明

本项目对应编译器设计专题实验一：DFA 模拟程序设计。

实验主要目标是通过程序实现 DFA 的输入、检查、运行与判定过程，加深对形式语言与自动机基本原理的理解。

在基础功能之外，本项目还补充实现了 DFA 可视化网页，并将代码整理上传至 GitHub，方便后续维护、展示与复现。

---

## 注意事项

1. `.dfa` 文件中字符集目前建议使用单字符输入符号。
2. 状态名称可以使用数字或字符串，例如 `1 2 3`、`q0 q1 q2`。
3. 转移矩阵的行数必须等于状态数。
4. 转移矩阵的列数必须等于字符集大小。
5. 转移矩阵中的每个状态必须属于状态集。
6. 空串在程序输出中使用 `ε` 表示。
